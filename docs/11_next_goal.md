# 11 · 下一阶段 Goal

## Goal

完成 2026-07-02 前端展示与对外沟通准备：把车载长期记忆 demo 包装成一套能同时面向技术评审、论文方向讨论、投资人/资源方沟通的材料，并在展示后进入可复现、可评测、可试点的下一阶段。

## Scope

本阶段优先服务四件事：

1. 前端能稳定展示长期记忆闭环：
   - `turn -> retrieve -> ask/act -> clarification -> learn -> summarize -> reuse`
2. 文档能讲清技术路线：
   - LightMem 检索
   - 结构化偏好表
   - 条件偏好激活
   - ACT / ASK 决策
   - 澄清学习
3. 文档能支撑论文/技术报告方向：
   - 任务定义
   - 研究问题
   - 评测指标
   - baseline / ablation 方向
4. 文档能支撑资源沟通：
   - 云服务和 GPU 推理
   - 数据集与标注
   - 前后端工程
   - 车载场景和试点合作

核心材料：

- `docs/13_showcase_technical_route_and_resource_plan.md`

## Acceptance

展示前：

1. 前端页面可打开并连接 API。
2. `一键全流程` 能完整跑通。
3. 偏好表能展示 `user_stated`、`offline_summary`、`learned_from_clarification` 等来源。
4. Decision Trace 能展示当前轮为什么 ASK / 为什么 ACT。
5. README 能清楚指向展示材料。
6. 展示材料包含：
   - 现场流程
   - 讲解词
   - 技术路线
   - 论文角度
   - 资源需求
   - PPT 结构
   - 风险边界

展示后 1-2 周：

1. 固化服务器部署方式，减少临场启动风险。
2. 增加 10 个稳定车载场景脚本。
3. 开始整理条件偏好评测集。
4. 明确云服务、数据标注和车载接口合作需求。
5. 准备一版技术报告或论文初稿大纲。

## Non-goals

- 不在当前 demo 仓库里做完整论文实验管线。
- 不把 mock 车控直接包装成量产能力。
- 不承诺当前系统已经支持多用户、多车辆、多租户。
- 不为了展示而硬编码新的假场景；展示必须基于真实 API 和现有闭环。
