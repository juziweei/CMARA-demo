# CMARA Demo 前端

车载长期记忆系统的前端展示层，一个单页 HTML 应用。

## 快速开始

### 方式一：双击启动（Windows）
双击 `start.bat`，然后在浏览器中打开 http://localhost:8080

### 方式二：命令行启动
```bash
cd frontend
python -m http.server 8080
# 访问 http://localhost:8080
```

### 方式三：直接打开
用浏览器直接打开 `index.html`（不推荐，部分浏览器会限制 file:// 协议下的 fetch 请求）

## 连接后端

### 方案 A：服务器 API 直接暴露
如果服务器防火墙已开放 8010 端口，前端直接连接 `http://114.215.186.130:8010`

### 方案 B：SSH 隧道（推荐）
在本地终端建立 SSH 隧道：
```bash
ssh -p 57003 -L 8010:127.0.0.1:8010 root@114.215.186.130
```
然后前端连接 `http://127.0.0.1:8010`（默认配置）

### 方案 C：本地运行
如果后端在本地运行，连接 `http://127.0.0.1:8010`

## 功能区说明

### 聊天区
- 输入文本发送到 `POST /turn`
- 系统回复展示在聊天区
- 状态标签：**ACTED**（绿）= 已执行 / **NEEDS INPUT**（橙）= 等待追问
- 追问状态时输入框自动切换为"回答"模式，发送到 `POST /clarification`

### 偏好表面板（📋）
- 展示 `GET /preferences` 的全部结构化偏好
- 列：ID | 偏好名 | 值 | 条件 | 状态(active/expired) | 来源
- 实时更新

### 决策 Trace 面板（🔍）
- 最近一轮的完整决策过程：
  - 决策结果（ACT/ASK、工具、参数、理由）
  - 上下文分析（维度提取）
  - 召回的偏好（含匹配分数）
  - LightMem 命中记录
  - 工具执行结果
- 可展开查看原始 JSON

### 学习记录面板（🧠）
- 展示通过追问学到的新偏好
- 展示因冲突而过期的旧偏好
- 展示离线总结抽取的偏好
- 按时间倒序排列

### 快速演示（▶）
- 自动运行完整的家庭出游场景：
  1. Day 1: 多轮对话 → 学到初始偏好
  2. 离线总结 → 抽取结构化偏好
  3. Day 2: "好热啊" → 系统发现冲突 → 追问用户
  4. Day 2: 用户回答 → 执行动作 → 学到新偏好 → 旧偏好过期
  5. Day 9: 同样场景 → 直接执行（不再追问）→ 闭环验证

### 控制按钮
- **🔗 连接**: 检测服务器健康状态
- **▶ 快速演示**: 运行完整场景演示
- **📊 Summarize**: 触发离线偏好抽取
- **🔄 Reset**: 清空所有数据

## 技术说明

- 纯 HTML/CSS/JS，无框架依赖
- 单文件部署，可直接用浏览器打开
- 暗色主题（GitHub 风格配色）
- 后端 CORS 已配置 `Access-Control-Allow-Origin: *`
- 所有 API 请求固定 `session_id: "default"`
- 服务器地址自动保存到 localStorage

## 设计原则

这个前端的目标是：
> 把后端闭环变成一个能看懂、能演示、能解释系统为什么这么做的展示层

不是做一个复杂的车机 UI，而是让观者清楚看到：
1. 系统记住了什么（偏好表）
2. 系统为什么这样决策（Trace 面板）
3. 系统刚学到了什么（学习记录面板）
4. 闭环是否生效（快速演示）
